# Mental Models Summary Boxes

| 01 | Introduction to Mental Models |
| ----- | :---- |
| **Definition**  *Mental models are simplified frameworks we use to understand the world, make decisions, and solve problems more effectively.* |  |
| **How to Apply** Build a diverse toolkit of models from multiple disciplines , psychology, economics, physics, biology. Apply the most relevant model to the situation at hand rather than forcing one model onto every problem. |  |
| **▶ Try This**  Pick one new mental model each week, study it, and consciously look for one real situation where it applies. |  |

| 02 | First Principles Thinking |
| ----- | :---- |
| **Definition**  *Break a problem down to its most fundamental, indisputable truths, then reason upward from scratch rather than relying on analogy or convention.* |  |
| **How to Apply** List every assumption you hold about the problem. Challenge each one by asking 'Is this definitely true?' Discard inherited wisdom that cannot be verified, and rebuild your solution only from what survives. |  |
| **▶ Try This**  Take a belief you hold about your work or life and ask 'Why?' five times in a row until you hit bedrock fact. |  |

| 03 | Inversion |
| ----- | :---- |
| **Definition**  *Instead of asking how to achieve a goal, ask what would guarantee failure , then avoid those things.* |  |
| **How to Apply** Define the desired outcome clearly. Brainstorm every action, habit, or condition that would guarantee you never reach it. Eliminate or guard against those failure modes rather than only chasing success. |  |
| **▶ Try This**  Before your next project kick-off, write a "pre-mortem": assume it failed completely and list the causes. |  |

| 04 | Opportunity Cost |
| ----- | :---- |
| **Definition**  *Every choice you make implicitly rejects every alternative , the value of the best foregone alternative is your true cost.* |  |
| **How to Apply** Before committing to any decision, explicitly name the next-best alternative. Weigh the value of what you gain against the value of what you give up, not just the monetary or time price. |  |
| **▶ Try This**  Before your next big "yes," write down the single best thing you are saying "no" to and decide consciously. |  |

| 05 | Second-Order Thinking |
| ----- | :---- |
| **Definition**  *Think beyond the immediate consequence of a decision to the downstream effects it will trigger in turn.* |  |
| **How to Apply** Map your decision three steps deep: "If I do X, then Y happens , and if Y happens, then Z follows." Evaluate whether Z, not just Y, is what you actually want. |  |
| **▶ Try This**  For your next important decision, write out the chain: Action → 1st effect → 2nd effect → 3rd effect. |  |

| 06 | The Map is Not the Territory |
| ----- | :---- |
| **Definition**  *Any model, plan, or description of reality is a simplification , the real world always contains more detail and surprise than the map suggests.* |  |
| **How to Apply** Treat your plans and mental models as useful approximations, not ground truth. Actively seek out what your model omits. Update your map quickly when reality contradicts it rather than defending the map. |  |
| **▶ Try This**  Identify one assumption in a current plan and go verify it directly in the real world this week. |  |

| 07 | Circle of Competence |
| ----- | :---- |
| **Definition**  *Stick to decisions within the domain you genuinely understand deeply; know and respect the boundary of your expertise.* |  |
| **How to Apply** Draw three honest circles: what you know deeply (operate here), what you are learning (expand carefully), what is outside your knowledge (defer or avoid). Before taking action, ask: which circle is this in? |  |
| **▶ Try This**  Write down your three genuine areas of deep expertise and explicitly label your next big decision as inside or outside them. |  |

| 08 | Occam's Razor |
| ----- | :---- |
| **Definition**  *When two explanations fit the facts equally well, prefer the simpler one , complexity is not a feature, it is a cost.* |  |
| **How to Apply** Start with the simplest possible explanation or solution. Add complexity only if the simple version demonstrably fails to explain the evidence or solve the problem. |  |
| **▶ Try This**  Look at a complex solution you are building and ask: what is the minimum version that still works? |  |

| 09 | The Lindy Effect |
| ----- | :---- |
| **Definition**  *The longer something non-perishable has already survived, the longer it is likely to continue to survive in the future.* |  |
| **How to Apply** When evaluating ideas, technologies, or practices, favour those with long track records over new entrants, especially in high-stakes domains. A book read for 500 years will likely be read for 500 more. |  |
| **▶ Try This**  Before adopting a new tool or methodology, ask how old the underlying principle is , not just the product. |  |

| 10 | Sunk Cost Fallacy |
| ----- | :---- |
| **Definition**  *Past investment , time, money, or effort already spent , is gone regardless of what you do next and should never drive future decisions.* |  |
| **How to Apply** Separate what you have spent from what you stand to gain going forward. Ask: "If I had not already invested in this, would I choose to start it today?" If no, stop. |  |
| **▶ Try This**  List one project you are continuing mainly because of what you have already put in , then honestly evaluate its future value. |  |

| 11 | The Pareto Principle (80/20 Rule) |
| ----- | :---- |
| **Definition**  *Roughly 80% of outcomes come from 20% of causes , a small number of inputs drive the vast majority of results.* |  |
| **How to Apply** Track your activities for one week and identify which 20% produce 80% of your meaningful results. Ruthlessly protect time for those activities and minimise or delegate the rest. |  |
| **▶ Try This**  List your top 10 tasks and circle the two that drive most of your results , then schedule those first tomorrow. |  |

| 12 | Cognitive Biases |
| ----- | :---- |
| **Definition**  *Systematic errors in thinking that cause us to deviate from rational judgment, usually without awareness.* |  |
| **How to Apply** Learn the most common biases (anchoring, availability, confirmation, etc.) and build check-in questions into important decisions. Slow down deliberate thinking when stakes are high. |  |
| **▶ Try This**  Before your next major decision, name one bias most likely to distort your thinking and design a simple counter-check. |  |

| 13 | Probabilistic Thinking |
| ----- | :---- |
| **Definition**  *Think in likelihoods and ranges rather than certainties , assign probabilities to outcomes rather than treating one scenario as inevitable.* |  |
| **How to Apply** For any forecast, identify the best case, worst case, and most likely case with rough probability weights. Update your probabilities as new evidence arrives rather than locking in an early view. |  |
| **▶ Try This**  For your next big decision, explicitly write: "I think there is a \_\_% chance this works, \_\_% it partially works, \_\_% it fails." |  |

| 14 | The Dunning-Kruger Effect |
| ----- | :---- |
| **Definition**  *People with limited knowledge in a domain tend to overestimate their competence; true experts tend to underestimate it.* |  |
| **How to Apply** Be most sceptical of your own confidence when you are new to a subject. Seek expert review of your work. Interpret high certainty as a signal to double-check, not a green light. |  |
| **▶ Try This**  In a domain where you feel very confident, find one expert who disagrees with you and genuinely try to understand their view. |  |

| 15 | Hanlon's Razor |
| ----- | :---- |
| **Definition**  *Never attribute to malice what can be adequately explained by incompetence, ignorance, or accident.* |  |
| **How to Apply** When someone's action harms or frustrates you, consider the most charitable explanation first. Ask whether the person had the knowledge, intention, and motive to act maliciously before concluding they did. |  |
| **▶ Try This**  The next time you feel wronged, write two alternative explanations , one malicious, one accidental , and investigate before deciding which fits. |  |

| 16 | Systems Thinking |
| ----- | :---- |
| **Definition**  *See the world as interconnected systems with feedback loops rather than as isolated chains of cause and effect.* |  |
| **How to Apply** Map the key elements of a problem and draw the relationships between them, including feedback loops. Look for delays, reinforcing loops that amplify, and balancing loops that dampen , then intervene at high-leverage points. |  |
| **▶ Try This**  Draw a simple diagram of your biggest current problem showing at least three elements and how they influence each other. |  |

| 17 | The Availability Heuristic |
| ----- | :---- |
| **Definition**  *We judge the likelihood of events by how easily examples come to mind, causing vivid or recent events to feel far more probable than they are.* |  |
| **How to Apply** When assessing risk or frequency, deliberately seek out base-rate statistics rather than relying on memorable anecdotes. Ask: "What does the data say?" before asking: "What do I recall?" |  |
| **▶ Try This**  Identify one risk you worry about disproportionately and look up its actual statistical probability. |  |

| 18 | Skin in the Game |
| ----- | :---- |
| **Definition**  *Decision-makers should bear real consequences , upside and downside , for the outcomes of their choices.* |  |
| **How to Apply** Align incentives so that the people making recommendations also share in the results. When seeking advice, prefer those who have personal stakes in the outcome over pure observers. |  |
| **▶ Try This**  Before taking advice, ask: "What does this person lose if they are wrong?" |  |

| 19 | The Law of Diminishing Returns |
| ----- | :---- |
| **Definition**  *Beyond a certain point, adding more input yields progressively smaller increases in output , and eventually none at all.* |  |
| **How to Apply** Find the inflection point in any effort , the moment when additional time, money, or resources stop yielding meaningful gains. Reallocate resources to higher-return activities before that point rather than pushing past it. |  |
| **▶ Try This**  Identify one area where you are investing heavily and honestly ask whether the last 20% of effort is producing meaningful results. |  |

| 20 | Confirmation Bias |
| ----- | :---- |
| **Definition**  *We unconsciously seek, interpret, and remember information that confirms what we already believe.* |  |
| **How to Apply** Actively search for evidence that contradicts your current view. Assign a "devil's advocate" role in group decisions. Weight disconfirming data at least as heavily as confirming data. |  |
| **▶ Try This**  Read one article or talk to one person who holds the opposite view to yours on an important topic before finalising your position. |  |

| 21 | Falsifiability |
| ----- | :---- |
| **Definition**  *A claim is only scientifically meaningful if it is possible, in principle, to prove it wrong , unfalsifiable beliefs are beyond the reach of evidence.* |  |
| **How to Apply** For any belief or hypothesis, ask: "What evidence would convince me I am wrong?" If no such evidence exists, treat the belief as a matter of faith rather than fact. |  |
| **▶ Try This**  Pick one strongly held belief and define the specific observation that would cause you to abandon it. |  |

| 22 | Survivorship Bias |
| ----- | :---- |
| **Definition**  *We draw conclusions from visible successes while ignoring the invisible failures that never made it through the filter.* |  |
| **How to Apply** Whenever you study success stories, actively ask what happened to those who tried the same approach and failed. Look for the graveyard of failed attempts before modelling your strategy on the survivors. |  |
| **▶ Try This**  Find one success story you admire and research the failure rate of others who followed the same path. |  |

| 23 | The Peter Principle |
| ----- | :---- |
| **Definition**  *In hierarchies, people are promoted based on performance in their current role until they reach a level of incompetence , where they stay.* |  |
| **How to Apply** Promote people based on the skills required in the new role, not just their performance in the old one. Recognise that an excellent individual contributor may not be an effective manager. |  |
| **▶ Try This**  Evaluate your next promotion decision on the requirements of the destination role, not the track record in the current one. |  |

| 24 | Tragedy of the Commons |
| ----- | :---- |
| **Definition**  *Shared resources are depleted when individuals, acting in their own rational self-interest, collectively over-consume them.* |  |
| **How to Apply** Identify any shared resource , attention, budget, infrastructure , where individual incentives conflict with collective welfare. Design governance rules, quotas, or private ownership structures to align individual and group interests. |  |
| **▶ Try This**  Spot one shared resource in your team or organisation that is being over-used and propose one constraint or governance rule. |  |

| 25 | Reciprocity |
| ----- | :---- |
| **Definition**  *People feel a powerful obligation to return favours, gifts, and concessions , giving first creates a natural pull toward return.* |  |
| **How to Apply** Build relationships by giving value first without expectation. In negotiation, make a concession and the other party will feel compelled to reciprocate. Be aware that reciprocity can also be weaponised through unsolicited gifts. |  |
| **▶ Try This**  Offer genuine help to one person in your network this week with no immediate ask attached. |  |

| 26 | Thought Experiment |
| ----- | :---- |
| **Definition**  *Explore hypothetical scenarios in the mind to test principles, stress-test beliefs, or generate insight without real-world experimentation.* |  |
| **How to Apply** Construct a simplified imaginary scenario that isolates the key variable you want to understand. Reason through the scenario rigorously. Use the conclusion to inform your real-world view or decision. |  |
| **▶ Try This**  Ask yourself: "What would happen if I doubled my price / halved my team / removed this feature?" and reason it through completely. |  |

| 27 | Relativity |
| ----- | :---- |
| **Definition**  *Observations of speed, position, time, and many human judgments are not absolute , they depend entirely on the observer's frame of reference.* |  |
| **How to Apply** When evaluating a situation, identify what frame of reference you are using and how it differs from others involved. Deliberately adopt another vantage point to test whether your conclusions hold. |  |
| **▶ Try This**  In a current disagreement, write a full description of the situation from the other party's perspective before responding. |  |

| 28 | Thermodynamics |
| ----- | :---- |
| **Definition**  *Closed systems naturally drift toward disorder (entropy) over time; maintaining order requires continuous energy input.* |  |
| **How to Apply** Apply this to organisations, relationships, and projects: without deliberate maintenance and investment, they degrade. Build regular "energy inputs" , reviews, re-alignment sessions, maintenance cycles , into every system you run. |  |
| **▶ Try This**  Identify one system in your life or work that is drifting toward disorder and schedule a deliberate maintenance action. |  |

| 29 | Inertia |
| ----- | :---- |
| **Definition**  *Objects , and organisations, habits, and people , tend to remain in their current state until acted upon by a sufficient external force.* |  |
| **How to Apply** To create change, apply enough concentrated force to overcome initial resistance rather than spreading effort thinly. To prevent unwanted change, build in stabilising structures that resist outside disruption. |  |
| **▶ Try This**  Identify one habit you want to change and design one specific, high-force trigger to overcome the inertia of the current state. |  |

| 30 | Friction and Viscosity |
| ----- | :---- |
| **Definition**  *Resistance embedded in a system , whether physical, procedural, or psychological , slows the flow of effort and output.* |  |
| **How to Apply** Map the friction points in any process: unnecessary approvals, unclear steps, tool-switching costs. Eliminate or reduce each one. Conversely, add deliberate friction to behaviours you want to discourage. |  |
| **▶ Try This**  Count the steps in one recurring process and remove at least one step that adds no value. |  |

| 31 | Velocity |
| ----- | :---- |
| **Definition**  *Unlike speed, velocity includes direction , moving fast in the wrong direction is worse than moving slowly in the right one.* |  |
| **How to Apply** Define both the rate of progress and the direction before optimising for speed. Regularly confirm that your fast-moving efforts are still pointed at the right goal; adjust direction before accelerating further. |  |
| **▶ Try This**  Ask of your current most urgent project: "Are we going fast in the right direction, or just going fast?" |  |

| 32 | Leverage |
| ----- | :---- |
| **Definition**  *A small, well-placed force applied at the right point can move a load far beyond what direct effort alone could achieve.* |  |
| **How to Apply** Find the highest-leverage point in any system , a key relationship, a bottleneck process, a critical assumption. Apply disproportionate effort or resources there rather than distributing it evenly. |  |
| **▶ Try This**  List your five current activities and mark the one that unlocks the most value for others , invest there first tomorrow. |  |

| 33 | Activation Energy |
| ----- | :---- |
| **Definition**  *Every process requires an initial energy input to get started , lower that barrier and the reaction proceeds naturally.* |  |
| **How to Apply** Identify what makes starting hard , ambiguity, setup cost, emotional resistance , and reduce it. Make the first step trivially easy. In habit formation, design the environment so the activation cost of the desired behaviour is near zero. |  |
| **▶ Try This: Reduce** the first step of one important habit to under two minutes so it requires almost no activation energy. |  |

| 34 | Alloying |
| ----- | :---- |
| **Definition**  *Combining two or more different materials or disciplines creates something stronger than any single element alone.* |  |
| **How to Apply** Deliberately mix expertise, perspectives, or resources from different domains. The strongest teams, products, and ideas often emerge from cross-disciplinary combinations that create properties neither element possesses individually. |  |
| **▶ Try This**  Bring one person from a completely different field into your next brainstorm and listen for how their framing changes your thinking. |  |

| 35 | Catalysts |
| ----- | :---- |
| **Definition**  *A catalyst accelerates a process without being consumed , it lowers the barrier to reaction without changing the start or end state.* |  |
| **How to Apply** Look for people, tools, or conditions that accelerate progress without requiring proportional ongoing effort. Invest in catalytic elements , the right introduction, the right software, the right framing , that make everything else flow faster. |  |
| **▶ Try This**  Identify one bottleneck in a current project and ask: "What one person, tool, or change would make this much easier without me carrying it?" |  |

| 36 | Evolutionary Natural Selection |
| ----- | :---- |
| **Definition**  *Variation plus differential survival equals adaptation , entities best suited to their environment reproduce and persist while others fade.* |  |
| **How to Apply** Run many small experiments (variation), measure results honestly (selection), and double down on what survives. Apply this to products, strategies, and habits: create options, test, keep what thrives, discard what doesn't. |  |
| **▶ Try This**  Run three small, low-cost experiments on a current challenge this month and kill the two that perform worst. |  |

| 37 | Replication |
| ----- | :---- |
| **Definition**  *The ability to copy a successful pattern faithfully , and the recognition that imperfect copying introduces variation that can drive improvement.* |  |
| **How to Apply** Document successful processes so they can be replicated consistently. Allow enough variation in replication to surface improvements. Distinguish between errors (fix them) and productive mutations (explore them). |  |
| **▶ Try This**  Document one process you do repeatedly so precisely that someone else could replicate it with zero guidance from you. |  |

| 38 | Ecosystems |
| ----- | :---- |
| **Definition**  *A community of interdependent elements , organisms, organisations, or ideas , that sustain each other through flows of energy, information, or value.* |  |
| **How to Apply** Map the ecosystem around your product, team, or community: who produces, who consumes, who decomposes and recycles. Design for the health of the whole system, not just your node within it. |  |
| **▶ Try This**  Draw a simple map of five entities that your work depends on and identify the most fragile link in that web. |  |

| 39 | Niches |
| ----- | :---- |
| **Definition**  *In any ecosystem, lasting success comes from finding and occupying a distinct role that others are not filling , not from competing head-on for the same resources.* |  |
| **How to Apply** Define the specific, narrow problem you solve better than anyone else. Resist pressure to compete in every dimension. Depth in a niche creates a durable advantage that broad competition erodes. |  |
| **▶ Try This**  Write a one-sentence description of the precise niche your work occupies , if it sounds like everyone else, sharpen it. |  |
